---

### 🐞 Кейс 1: Ошибка в логике статусов (критический дефект)

**Ситуация:**  
В системе использовался workflow из ~20 статусов. До 17 статуса карточка находилась у менеджера по закупкам, затем переходила контент-менеджеру для публикации.  

**Действие:**  
Проверила сценарий возврата карточки на предыдущий статус после публикации.  

**Проблема:**  
Возврат был возможен только через снятие с публикации → товар исчезал с сайта.  

**Риск:**  
Потеря продаж из-за недоступности товара.  

**Результат:**  
✔ Выявлен критический дефект в бизнес-логике  
✔ Баг задокументирован и передан в разработку  

---

### 🐞 Кейс 2: Неэффективный процесс обновления карточек  

**Ситуация:**  
Для обновления данных требовалось пересохранять родительскую карточку и все варианты (до 20).  

**Проблема:**  
Избыточные действия → увеличение времени обработки.  

**Решение:**  
Предложено реализовать обновление через одну точку — родительскую карточку.  

**Результат:**  
✔ Упрощён пользовательский сценарий  
✔ Сокращено время работы контент-менеджеров
